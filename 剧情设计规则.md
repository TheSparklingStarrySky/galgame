# 《零点协议》剧情设计规则

本文档用于约束《零点协议》的剧情扩写、分支设计与文本实现。目标不是单纯增加字数，而是让玩家在每个场景中获得新的事实、关系变化或危险判断。

## 一、核心体验

1. **先求证，再相信**：角色面对绑架、项圈和死亡游戏时，应先提出正常人的解释并尝试验证，不能因为类型惯例而立刻接受规则。
2. **规则制造关系**：生还条款不只是谜题条件，还必须迫使角色合作、隐瞒、交换或互相防备。
3. **死亡改变行为**：每次死亡都要改变幸存者的语言、站位、触碰物品的方式和后续决策，不能只作为转场。
4. **信息可以复查**：关键推理必须由玩家见过的规则、证据和人物行为组成，不依赖结局前突然出现的新设定。
5. **恋爱来自共同经历**：感情线通过共同行动、相互照顾和承担风险建立，不靠一次好感选项直接确认关系。

## 二、场景的最低构成

每个剧情场景至少完成以下三项中的两项：

- **新事实**：地点、规则、人物经历或证据出现可验证的新信息。
- **新立场**：至少一名角色改变合作、怀疑、隐瞒或行动方向。
- **新代价**：玩家或角色失去时间、安全、信任、资源或退路。

场景编写前使用以下模板：

| 项目 | 必须回答的问题 |
| --- | --- |
| 场景目标 | 玩家进入场景时想弄清什么？ |
| 阻力 | 谁或什么阻止目标立即达成？ |
| 可见动作 | 角色在说话之外具体做了什么？ |
| 信息变化 | 场景结束时，玩家比开始多知道什么？ |
| 关系变化 | 谁更信任谁，谁开始防备谁？ |
| 退出钩子 | 为什么玩家需要进入下一场景？ |

纯解释场景也必须存在打断、质疑或验证动作，避免所有角色轮流朗读设定。

## 三、节奏与现实接受过程

角色遭遇异常事件时，按以下顺序推进：

1. **感官确认**：确认身体、时间、空间和随身物品。
2. **普通解释**：优先考虑绑架、勒索、恶作剧、事故或非法实验。
3. **低风险试探**：试门、检查通讯、观察摄像头、交换失忆前经历。
4. **相互质疑**：怀疑内应、演员、身份造假或共同仇家。
5. **不可逆证据**：只有死亡、真实伤害或无法伪造的规则反馈，才能迫使多数人接受死亡游戏的现实。
6. **余震**：否认、愤怒、呕吐、僵住、过度理性化等反应必须保留人物差异。

不允许使用“大家很快冷静下来”“众人接受了现状”等概括跳过过程。

## 四、群像角色登场规则

玩家在同一场景中能够看见的正式参与者，首次登场必须至少具备：

1. 一个能被立绘识别的外貌或职业道具。
2. 一次由本人说出的姓名、编号和身份。
3. 一个体现性格的动作习惯。
4. 一个与当前困境有关的判断或诉求。
5. 一项可以在后续回收的能力、恐惧、秘密或错误认知。

禁止把正式角色压缩成“剩下的人依次介绍了自己”。需要延迟登场时，必须给出场内原因，例如缺席、失踪、拒绝回应、身份被篡改或视角无法接触；延迟本身必须成为可追查的悬念。

### 第一章参与者职责

| 编号 | 首章功能 | 首次回收方向 |
| --- | --- | --- |
| 01 沈砚 | 玩家视角与事故推理 | 组织共同证词 |
| 02 黎星遥 | 通信与信号验证 | 中继器、第二网络 |
| 03 苏弥 | 医疗判断与存活底线 | 处决方式、死亡时间 |
| 04 韩骐 | 逃生能力与强制秩序 | 终端占有冲突 |
| 05 吴峥 | 对暴力破局的错误信心 | 首次处决演示 |
| 06 唐弈 | 将死亡视作概率和利益 | 精确四人生还条款 |
| 07 林澄 | 空间记忆与被忽视的观察 | 备用通道、脚步记录 |
| 08 陈默 | 系统权限与界面异常 | 终端权限、日志删除 |
| 09 高原 | 建筑设备与声音判断 | 通风、设备间、振动 |
| 10 周叙 | 普通人的否认与家庭牵挂 | 第二例死亡的情感重量 |
| 11 叶岚 | 群体状态观察与冲突降温 | 恐慌传播、证言偏差 |
| 12 未知 | 缺席本身就是线索 | “全员到齐”与空名册的矛盾 |

### 第二章回收要求

- 封锁不是单纯倒计时：必须改变可用地图、撤离路径与角色站位。
- “留下一个人”先作为看似合理的救援前提出现，再由控制箱、制动索和空底座三项证据拆解，不能靠主角灵光一现取消代价。
- 08 号陈默的隐瞒必须同时具有合理恐惧和现实后果；公开操作记录只能降低嫌疑，不能直接证明无罪。
- 09 号高原需要通过设备声音、断口与机械限制提供知识，但最终的 12 号结论仍由玩家组合证据得出。
- 12 号在第二章只验证“身份槽能够脱离真人存在”，不提前揭晓幕后操作者。
- 三条感情线必须要求第一章同行与二次调查选择一致，第二章的共同留守和事后照顾才会开放。

## 五、对白、旁白与人物声音

- **对白**只写角色实际说出口的话，并显示角色姓名。
- **旁白**负责环境、动作、心理、推理和第一人称叙述，不显示姓名。
- 只要一句内容同时满足“由明确角色提出”且“会改变争论、分工或信任”，就必须优先让该角色亲口说出，不能用“某人提议、某人反对、大家讨论后决定”概括。
- 多人争论至少保留主张、反对和回应三个声音。旁白可以写停顿、视线、站位和未说出口的判断，但不能替三个人完成整场辩论。
- 专业结论应拆成“观察者报告事实、另一人提出普通解释、专业角色设计反证、众人确认边界”，避免专业角色一次说完谜底。
- 并非所有叙述都要改成对白。第一人称心理、环境压力、动作矛盾和玩家尚不能公开的推理继续使用旁白。
- 同一剧情节点中切换说话者时，必须拆成独立段落并设置对应 `passageSpeakers`。
- 角色立绘可以在旁白段落继续留场，但姓名栏不得继承上一段角色。
- 每名主要角色应有稳定的语言习惯，不能共享同一种“冷静分析”口吻。

角色声音基准：

- 沈砚先陈述事实，再给判断；紧张时会把情绪转译成证据。
- 黎星遥使用设备、频率、连接等具体词汇，不轻易作人格判断。
- 苏弥先确认风险与身体状态，不用空泛安慰替代医学判断。
- 韩骐使用命令句和行动方案，信任来自可执行的纪律。
- 唐弈用概率、下注和交换描述关系，但不写成无缘由的疯子。
- 林澄说话谨慎，会明确区分“我看见的”和“我猜的”。
- 陈默关注权限、接口和系统是否允许某个操作，紧张时会咬指甲。
- 高原先听、摸、测量，再开口；结论短而具体。
- 周叙倾向用数字和常识否认失控，越恐惧越强调自己只是普通人。
- 叶岚描述可观察行为，不随意诊断陌生人，也不能充当作者的读心工具。

提交章节前运行 dart run tool/story_dialogue_audit.dart。工具用于抓取“旁白中出现发言动词、却没有直接对白”的可疑段落；报告是人工复核入口，不是把所有命中项机械改成台词的硬指标。

## 六、信息揭示与悬念

每项重要信息分为三层：

1. **迹象**：玩家先看到异常，但不知道含义。
2. **解释**：角色提出至少两个竞争假说。
3. **验证**：通过行动、规则反馈或证据排除错误解释。

延迟信息必须满足：玩家知道“这里存在一个空白”，并知道当前为什么无法获得答案。不能等到后续章节才声称某个未出现的人一直在场。

12 号采用明确的缺席悬念：集合厅只有十一名可见参与者，第十二把椅子为空，名册显示“无记录”，但主办方仍宣告“全员到齐”，总人数也按十二人计算。

## 七、选择与分支

每个选择至少影响一种后续内容：

- 可见行动或台词；
- 合作、逻辑或人物信任数值；
- 可取得的证据；
- 他人对玩家的称呼、站位或信息公开程度；
- 线路图上的后续节点或结局资格。

选项文字要表达玩家真正采取的行动，说明文字表达风险。禁止使用含义不清的“相信她”“保持沉默”而不说明对象和代价。

## 八、推理与规则杀人

- 规则文本、现场事实和设备反馈必须彼此可对照。
- 真相所需的关键证据在推理前全部出现。
- 红鲱鱼应来自人物的合理误解或利益冲突，不靠伪造叙述视角。
- 专业角色可以提供知识，但不能直接替玩家完成结论。
- PDA 地图负责空间与行动规划，剧情线路图负责选择与后果，两者不能互相替代。
- 调查场景只在可搜索位置显示亮点，不把物品图、名称或正式线索悬浮在背景上。点击后物品才以可识别图片进入背包。
- 每个调查房间至少提供三个核心对象和一个环境对象。环境对象可以提供工具、旁证或排除错误方向，不能让所有可点击物都恰好等于正确答案。
- 点击对象只进入检查台，不直接取得完整线索。玩家要在观察、测量、触摸、拆解、离线复现、比对样本等动作中选择。
- 正式线索至少需要两个前置事实，其中一个应来自另一对象或工具。例如先观察中继器余温，再把绝缘拨片从背包拖到黑盒上；先隔离 12 号底座网络，再把离线测试线拖到底座上。
- 每件核心物品按“普通外观、局部异常、无恶意解释、反证操作、可复查结论”五层展开。错误检查可以排除一个方向，但不能伪造线索或无端惩罚玩家。
- 背包默认收起，通过调查界面右侧独立按钮打开；不得长期遮挡场景，也不得把背包内容塞进正文文本框。
- 组合提示只能说明“可以尝试组合”，不能提前写出正确工具名称。缺少前置观察或道具时使用中性反馈，不替玩家公布答案。
- 观察、拆解与组合会形成永久物品状态。详情页重新打开、离开地图或读档后，必须显示该状态对应的新名称、图片和描述，不能退回初始提示。
- 一次性钥匙、拨片、耗材和被开启容器在成功使用后从背包移除；组合产物必须拥有独立物品 ID、图片和描述，不能只用一行系统文字代替。
- 调查完成条件是核心结论全部经过验证，不是点击次数达到数量。背包物品、物品状态、组合记录与阶段线索必须可存档，支持离开地图后再返回使用早先取得的道具。
- 解密至少包含两个互相支持的步骤，例如先校准频率，再根据波形规律选择同步相位。
- 最终推理先要求玩家把证据放入“现场事实、规则阈值、实施媒介”等论证位置。错误组合应允许留在当前页重组；证据链闭合后才开放假说提交，不能只从答案列表中猜一个结论。

## 九、感情线

每条感情路线按以下顺序发展：

1. 因规则或任务同行。
2. 发现对方的能力与边界。
3. 看见对方不愿公开的脆弱点。
4. 玩家承担一次具体风险或责任。
5. 对方在主线推理中交出只有信任才能提供的证据。
6. 结局确认关系变化，但不能抹去死亡游戏的创伤和现实后果。

林澄已满 18 岁，其路线保持青涩、克制和非性化表达，核心是平等对待、共同生还与毕业后的约定。

## 十、结局收束

- 结局判定后先进入正常对话演出，再进入结局结算页。
- 每个结局至少回答：玩家的判断造成了什么后果、幸存者关系如何变化、七天规则如何结束、离开设施后留下什么创伤或约定。
- 坏结局不能只写“你失败了”，必须展示错误判断如何逐步破坏合作。
- 人物结局必须回收该路线的能力、脆弱点和共同证据，恋爱确认只能发生在这些经历之后。

## 十一、实现与验收

- `StoryBeat` 是存档、历史与文本推进节点，不因单纯分页重复创建。
- 剧情线路图只保留规则揭示、死亡、关键选择、调查、推理和结局等里程碑；普通对话段落不显示为线路节点。
- 同一节点内的对白与旁白使用 `passageSpeakers` 逐段标记。
- 角色节点通过 `portraitMood` 选择与当前动作、情绪一致的差分立绘。
- 正文优先按完整句分页，固定文本框内不滚动。
- 同一地点连续承担两段以上独立场景，或其空间结构会影响路线、证据与危险判断时，应配置独立背景，不能长期借用泛用走廊。
- 当角色发生会持续影响后续演出的身体状态、动作状态或关系状态时，应新增差分立绘。例如受伤夹板、公开对峙、疲惫放松；一次性轻微皱眉可复用现有差分。
- 选项、调查结果和系统面板不得进入正文文本框。
- 新增可见角色必须同时完成：`Speaker`、姓名、颜色、立绘路径、剧情节点、PDA 解锁条件和测试。
- 新增关键选择或结局时检查线路图连接；新增普通对白时不扩充线路图。

章节提交前检查：

- 可见参与者是否都完成了有效登场？
- 旁白是否错误挂在角色姓名下？
- 是否有旁白概括了本应由两名以上角色演出的争论？
- 角色是否过快接受异常规则？
- 每次死亡是否改变了后续行为？
- 每个选择是否至少产生一个可见后果？
- 谜底是否能由已经展示的证据推出？
- 调查热点是否在未点击前就泄露了线索名称？
- 每条正式线索是否至少经过一次观察和一次验证，而非点一下直接获得？
- 组合界面是否泄露了正确工具名称，物品再次打开时是否保留了最新图片与描述？
- 一次性物品是否在成功使用后消失，新产物是否有独立图片与说明？
- 关系线是否通过共同经历而不是单次选项推进？
- 12 号等延迟信息是否具有明确、可追查的缺席理由？

## 十二、体量与章节验收

完整游戏目标为 120,000 至 160,000 个独立可见字符，单周目正文目标为 80,000 至 100,000 个字符。目标用于防止章节过度压缩，不允许用重复解释、无后果闲聊或拆碎节点机械凑字数。

章节正文原则上保持以下体量：

- 普通章节 12,000 至 20,000 个可见字符。
- 终章 18,000 至 24,000 个可见字符。
- 章节坏结局 1,500 至 3,000 个可见字符。
- 普通结局 3,000 至 5,000 个可见字符。
- 人物路线结局 4,000 至 7,000 个可见字符。
- 共同线真结局 6,000 至 10,000 个可见字符。

每章交付时同时记录四项数据：全部新增正文字符、单条公共路线字符、正文节点数量、线路图重要节点数量。正文增长与线路图增长必须分开衡量。

章节从开场到结束至少包含：

1. 一段让角色处理上一章后果的低压场景。
2. 一项逐步形成的新规则压力或资源冲突。
3. 一次玩家可操作的调查、解密、地图规划、托管或表决。
4. 一场由个人条款或旧选择引发的人际冲突。
5. 一项能够由已有证据验证的章节结论。
6. 一段余震与次章钩子，不能在解谜成功后立即显示章节结束。

完整体量预算、七日八章结构和作者侧谜底统一维护在[《剧情总规划》](剧情总规划.md)。如果后续剧情需要改变 12 号性质、周叙死亡责任或 ZERO 三层含义，必须先修改总规划并审计已经写入的伏笔。
