# 零点协议

一款使用 Flutter + Flame 制作的原创死亡游戏 Galgame。12 名参与者在一座与外界隔绝的旧企业研修设施中醒来，必须在七天内满足各自不同的“生还条款”。玩家需要确认现状、交换条件、验证谎言、规划路线，并识破利用通用规则完成的杀人手法。

## 已实现

- Flame 全屏二次元室内场景、环境光效与中继器波形
- 938 个可回看故事节点，当前数据约 14.70 万可见字符；第一至七章共 930 个正式节点，已覆盖规则公布、六日封锁、CASE 01 至 CASE 06、普通追猎与审计同步关闭
- 十名可见配角均有中立与剧情动作差分，包括 08–11 号独立登场和 18 岁双马尾 JK 林澄
- 连续三章共同开局：正确破解中继器案件后依次进入“封锁顺序”与“互为人质”；PDA 已显示公开主条件与后续加密校验字段
- 黎星遥、苏弥、林澄三条感情支线，独立信赖值、专属调查证据与第二章差分；完整人物结局按七日总规划重写
- 实体化 PDA 系统：设备外壳、状态灯、个人条款、12 人名册、区域状态、路线标记、证据库、审计记录与统一系统菜单
- Galgame 基础系统：8 槽存档与读档（手动存档同时保存当前进程画面缩略图）、自动存档、自动播放、已读快进、历史回顾、设置
- 收集系统：31 组共 62 帧事件 CG；正文由对应剧情节点精确切帧，完整看完末帧后解锁，鉴赏模式点击画面直接翻到组内下一帧；另有 6 个结局回顾条目
- 精简剧情线路图：938 个故事节点只显示 77 个规则、死亡、关键选择、调查、推理、封锁与结局里程碑
- 密室式现场调查：场景中只显示亮点，右侧按钮按需打开背包；组合提示不泄露正确工具，道具可拖放，也可选中后横向滚动并与远端物品组合；一次性物品会消耗，物品图片与描述状态随存档永久保留
- 分时段环境机关：跨区域门禁与天平称重用于首次调查，九宫格路由改接留到封锁危机，隐藏校验表则在脱险后复盘
- 已接入的早期坏结局会先播放收束剧情再进入结算页；旧版跨日概述结局只保留为主题草案，后续改写为完整七日路线
- 第一至七章已有完整可玩流程：第七章含 154 个节点、24,459 个正文字符、控制核心调查、CASE 06、三种追猎与二次死亡分支，以及九人同步关闭 ZERO 重放节点的审计行动线
- 七日双周目总规划：目标 17 至 22 万独立剧情字符；一周目从第三天起进入完整互杀线，完成七日普通结局后解锁二周目审计真相线
- 双周目状态已接入存档：七日普通结局解锁审计周目，第二章取得灰色索引，第三章恢复高危物资目录，第四章使用预演表重新封存六件物资；存活名单、死亡责任与物资状态均可存读档
- 正式资源按场景、角色差分、调查物品、BGM、音效和角色语音分类，旧版资源移入 `assets/archive/v1`
- 沉浸式横屏界面，支持刘海区域全屏；播放控制、固定文本框和分支各自独立
- 长对话会优先在句末分页，点击继续显示同一节点的下一段文本

剧情文档：已实现内容见 [`剧情.md`](剧情.md)，七日八章、人物弧线、谜团递进和文本预算见 [`剧情总规划.md`](剧情总规划.md)，写作与实现标准见 [`剧情设计规则.md`](剧情设计规则.md)。

## 运行

```bash
flutter pub get
flutter run -d chrome
```

质量检查：

```bash
flutter analyze
flutter test
flutter build web
```

统计剧情体量：

```bash
dart run tool/story_metrics.dart
```
