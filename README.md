# 零点协议

一款使用 Flutter + Flame 制作的原创死亡游戏 Galgame。12 名参与者在一座与外界隔绝的旧企业研修设施中醒来，必须在七天内满足各自不同的“生还条款”。玩家需要确认现状、交换条件、验证谎言、规划路线，并识破利用通用规则完成的杀人手法。

## 已实现

- Flame 全屏二次元室内场景、环境光效与中继器波形
- 243 个可回看文本节点，当前故事数据约 3.59 万可见字符：第一章完成规则公布、项圈处决与中继器案件，第二章继续推进首次永久封锁和 12 号空身份之谜
- 十名可见配角均有中立与剧情动作差分，包括 08–11 号独立登场和 18 岁双马尾 JK 林澄
- 连续两章主线：正确破解中继器案件后进入“封锁顺序”；错误推理与合作崩溃仍会进入坏结局
- 黎星遥、苏弥、林澄三条感情支线，独立信赖值、专属调查证据与第二章差分；完整人物结局按七日总规划重写
- 实体化 PDA 系统：设备外壳、状态灯、个人条款、12 人名册、区域状态、路线标记、证据库与统一系统菜单
- Galgame 基础系统：8 槽存档与读档、自动存档、自动播放、已读快进、历史回顾、设置
- 收集系统：CG 鉴赏、6 个结局回顾条目与待扩写的结局主题
- 精简剧情线路图：243 个正文节点只显示 32 个规则、死亡、关键选择、调查、推理、章节与结局里程碑
- 密室式现场调查：场景中只显示亮点，右侧按钮按需打开背包；组合提示不泄露正确工具，道具可拖放取证，一次性物品会消耗，物品图片与描述状态随存档永久保留
- 已接入的坏结局会先播放收束剧情再进入结算页，其余结局主题将在终章后重新接线
- 第一、二章完成扩写：关键会议与复盘改为角色直接对话，加入医务室、储藏间、档案区场景、R-08 时间线伏笔、三种体育馆路线与救援余波
- 七日八章总规划：目标 12 至 16 万独立剧情字符，单周目约 8 至 10 万字符，覆盖每天封锁、条款冲突、案件机制与完整结局
- 正式资源按场景、角色差分、调查物品、BGM、音效和角色语音分类，旧版资源移入 `assets/archive/v1`
- 沉浸式横屏界面，支持刘海区域全屏；播放控制、固定文本框和分支各自独立
- 长对话会优先在句末分页，点击继续显示同一节点的下一段文本

剧情文档：已实现内容见 [`剧情.md`](剧情.md)，七日八章、人物弧线、谜团递进和文本预算见 [`剧情总规划.md`](剧情总规划.md)，写作与实现标准见 [`剧情设计规则.md`](剧情设计规则.md)。

## 运行

```bash
flutter pub get
flutter run -d chrome
```

质量检查：

```bash
flutter analyze
flutter test
flutter build web
```

统计剧情体量：

```bash
dart run tool/story_metrics.dart
```
