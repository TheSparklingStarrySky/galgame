# 资源目录规范

当前游戏只从以下目录加载正式资源：

```text
assets/
├── audio/
│   ├── bgm/                 # 循环背景音乐
│   ├── sfx/                 # 界面、环境与剧情音效
│   ├── voice/               # voice/<角色ID>/<剧情节点>_<序号>.*
│   ├── audio_manifest.json   # 音频用途、循环点和台词映射
│   ├── pixabay_sources.json  # 已下载素材的作者、链接与许可检查记录
│   └── original_sources.json # 项目原创合成音频与生成器记录
├── images/
│   ├── characters/          # characters/<角色ID>/<表情或动作>.png
│   ├── cg/                  # cg/<事件ID>/<帧序号>.png
│   ├── items/               # items/<场景ID>/<可调查物品>.png
│   └── scenes/              # 横屏场景背景
└── archive/v1/              # 第一版未使用资源，不参与正式构建
```

## 命名规则

- 角色目录和文件名使用小写蛇形命名，例如 `li_xingyao/alarm.png`。
- 每名角色至少保留 `neutral.png`，剧情节点通过 `portraitMood` 选择差分。
- `characters/shen_yan/neutral.png` 是主角 CG 与特殊镜头的固定形象参考；第一人称普通对白不调用该立绘。
- 物品图使用透明 PNG，点击范围由 UI 控制，不把热点烘焙进背景。
- 事件 CG 使用 16:9 横屏图组，同一事件按 `01.png`、`02.png` 顺序编号；每帧必须同时包含事件动作、人物与环境，不得复用纯背景图。
- Pixabay 原始素材保留为 MP3，项目原创合成音频使用 AAC/M4A；两类运行时资源均已接入并通过解码检查。
- 音频需求、优先级、AI 提示词和授权留档流程见根目录 `音频规划.md`；机器可读清单维护在 `audio/audio_manifest.json`。
- 旧资源只能从 `archive` 中人工迁回，不允许业务代码直接引用归档目录。

## 差分与场景新增标准

- 同一地点连续承担两段以上剧情，且空间结构影响证据、路线或风险时，新增独立背景，不复用泛用走廊。
- 角色出现持续性身体变化或关键动作变化时新增差分；只改变一句台词的轻微表情可复用已有差分。
- 当前新增持续状态差分：`gao_yuan/injured.png`、`su_mi/relieved.png`、`su_mi/shaken.png`、`han_qi/conflicted.png`、`han_qi/armed.png`、`li_xingyao/relaxed.png`、`li_xingyao/vertigo.png`、`lin_cheng/anxious.png`、`chen_mo/guarded.png`、`chen_mo/desperate.png`。
- 背景保持 16:9 横屏、无人物、前景留出立绘空间；角色差分保持透明 PNG、完整头脚和统一人物比例。
- 新资源接入后同时更新 `SceneKey`、场景映射、`portraitMoods`、资源清单和存在性测试。

## 事件 CG 规则

- `StoryBeat.cgId + cgFrame` 只绑定实际发生该画面的正文节点。同一节点的正文分页不会切换 CG；只有剧情进入下一帧对应的节点时才换图，并隐藏普通站立立绘。
- 一组 CG 的末帧节点读完后才写入解锁记录。CG 鉴赏只显示玩家已在正文中看完整组的事件图。
- 鉴赏模式不重放正文；点击大图直接切换到组内下一帧，末帧后循环到第一帧。
- 当前接入十六组三十二帧：既有八组事件、第五章 E-04 身份复原，以及移动柜复仇、腐蚀剂灭口、服务器隔离、审计预防和三条人物关系事件。每一帧都绑定实际发生对应动作的正文节点，不在正文中自动连播。

## 调查物品展示规则

- 场景内只显示与背景融合的亮点，不在热点上显示物品图、物品名或线索结论。
- 点击亮点后，物品图才进入背包；背包默认收起，由调查界面右侧 `BAG` 按钮打开。点击背包物品可查看细节、取出内部工具或记录基础观察。
- 需要道具的检查不显示为可直接点击的结论按钮；玩家可以在工具与目标之间任选一个作为组合物，也可以将任意一方拖向另一方。横向滚动背包不会取消选中状态或重置当前位置。
- 未完成组合只提示“可尝试与其他物品组合”，不得直接显示正确工具、钥匙或门禁卡名称。
- 组合可以产生新物品、改变目标状态或验证线索；错误组合不消耗道具，只给出不适用反馈。
- 基础观察和成功组合必须永久更新目标的名称、图片或描述；重新打开背包、切换地图和读档后仍显示最新状态。
- 一次性物品成功使用后从背包移除。若钥匙与箱子都属于一次性对象，开箱时同时移除两者，并以独立图片、名称和描述加入箱内新物品。
- 背包物品、已执行组合和阶段线索必须进入存档快照，以支持跨地图获得门禁卡、返回原地开门等谜题。
- 一个调查房间至少包含一个工具或环境对象，避免所有亮点都与最终答案一一对应。
