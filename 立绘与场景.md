# 《零点协议》美术规范

## 整体方向

作品采用现代日系动画 Galgame 的二次元视觉：清晰线稿、细腻赛璐与手绘室内背景，同时保留成熟、克制的封闭设施求生气质。舞台是经过秘密改造的旧企业研修中心，不是轮船、海上平台或光洁的太空舱。科技感来自具体的门禁、广播、监控、无线电设备和项圈读数。

配色以石墨黑、设施绿、旧象牙白、氧化红和琥珀色构成。琥珀色表示可操作的规则与倒计时，设施绿表示已验证信息，红色只用于处决、区域封锁和其他不可逆状态。

## 角色立绘

立绘统一使用现代日系视觉小说画风、清晰动画线条、表现力明确的眼神和干净赛璐。服装保留职业使用痕迹，角色差异优先来自剪影、姿态、工具和 PDA 持有方式，不只依靠发色。

## 核心场景

### 单人宿舍

已接入：`assets/images/scenes/dormitory_room.png`

无窗、单人床、钢制书桌、停摆的时钟与无法主动呼叫的墙面广播。房间应让人先怀疑绑架与非法拘禁，而不是立刻联想到死亡游戏。

### 宿舍走廊

已接入：`assets/images/scenes/facility_corridor.png`

旧式企业设施走廊，两侧房门、外露管线、摄像头和尽头的防火门构成明确纵深。地面保持干燥，不出现海水、雨幕或船舶元素。

### 集合厅

已接入：`assets/images/scenes/assembly_hall.png`

十二把不完全相同的折叠椅围成松散圆形，墙上黑屏与扬声器由主办方控制。场景中央留出人物立绘和规则演示所需空间。

### 监控室

已接入：`assets/images/scenes/control_room.png`

老式 CRT 与新式显示器混用，无对外窗户。工作台上的便携中继器、监控画面和机柜作为调查热点。

## UI 规范

- 对话框与系统面板是工具表面，圆角不超过 6px；PDA 使用独立设备外壳、状态灯、屏幕边框与扫描线。
- 桌面端对话框最大宽度 1120px；移动端保留不少于 16px 安全边距。
- 可交互规则使用琥珀色，已验证证据使用设施绿，危险状态使用氧化红。
- PDA 区域地图与剧情线路图使用不同图标和空间尺度，避免玩家混淆。
- 游戏画面以横屏为唯一交互基准；自动播放、快进与 PDA 固定在右侧控制栏。
- 底部文本区高度固定为 112px，只承担说话者、正文与推进状态；长文本按句末分页，不在框内滚动。
- 剧情选项使用文本区上方的独立浮层，不进入文本框。
- 角色立绘位于场景层与选项/文本层之间，下半身可以被不透明文本框自然遮挡，不得覆盖右侧控制栏。
- 角色节点使用 `portraitMood` 选择表情或动作差分；不存在的差分必须回退到 `neutral`，不能临时拉伸或镜像伪装成新动作。
- 存档、读档、历史回顾、线路图、设置与收集入口集中在 PDA “系统”页，不占用游戏 HUD。
